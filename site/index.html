What it does

A full-featured data source, not a thin HTTP wrapper.

✍️

Write DQL

A Monaco editor with DQL syntax highlighting, one-keystroke formatting, and autocomplete backed by your live Grail schema — or build queries visually with the Builder/Code toggle.

📊

Metrics, logs & tables

Timeseries and table result shapes, real Grail timeframe + interval series, unit and display-name field config from labels, plus a logs view with derived fields, a log-volume histogram, and log-row context.

🔗

Distributed traces

A clickable trace list, full OpenTelemetry-compatible span detail, and trace-to-logs / trace-to-metrics correlation back into Grail.

🧩

Works everywhere

Alert rules, annotations, template/variable queries, and ad-hoc filters — all sharing the same server-side macro expansion, so an alert evaluates over its own window exactly like the panel it came from.

⚙️

Production backend

A Go backend with retry-and-backoff on 429/5xx (honoring Retry-After), a per-instance concurrency cap, request-context cancellation, and Prometheus self-metrics.

🔁

Coming from PromQL or SQL?

DQL is pipeline-based and readable. The README maps common PromQL and SQL patterns to their DQL equivalents to get you productive fast.

Quick start

Install, connect a tenant, and write your first query.

Install

Once available in the Grafana plugin catalog: 

grafana-cli plugins install \
  discostu105-grail-datasource

Then restart Grafana and add the data source. For a self-hosted or locally built copy, see CONTRIBUTING.md.

Requirements

  • A Dynatrace platform tenant — the https://<env>.apps.dynatrace.com host (not the classic *.live.dynatrace.com host).
  • A platform token (dt0s16.*) with the scopes for the data you query.
  • Grafana ≥ 12.3.0.
